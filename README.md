QUESTION: 
How can we design an AI-powered backup system that reduces backup
time by efficiently detecting changes, eliminating redundancy, and
optimizing data transfer?
REAL-WORLD ANALOGY:
Imagine backing up your laptop daily. Instead of copying all files every
time, you only copy the files that changed, making the process much
faster and efficient.
PROBLEM STATEMENT:
Design a system which optimizes backup processes
This system is supposed to detect changes in files using hashing
techniques
This system is supposed to perform incremental backups instead of full
backups
This system is supposed to eliminate duplicate data using
deduplication techniques
This system is supposed to upload data in parallel to improve speed
This system is supposed to prioritize critical data using AI-based
decision making
The system is supposed to minimize backup time and storage usage
The key feature of the system is fast, efficient, and resource-optimized
backups
ANY AI MODELS OR CLOUD STORAGE SERVICES CAN BE USED
(Examples: Azure Blob Storage, AWS S3, Python threading)
